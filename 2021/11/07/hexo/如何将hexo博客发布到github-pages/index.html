如何将 Hexo 博客发布到 GitHub Pages

发表于 2021-11-07 更新于 2023-01-19 Disqus：

具体的过程详见 Hexo 官方的教程。这里主要说几点实践的时候容易搞不清楚的。

如果想将博客部署到 GitHub Pages，推荐安装 hexo-deployer-git。

然后关于博客的链接，比如你的 GitHub 账号是 AAAA，那么你的 GitHub 主页链接应当是 https://github.com/AAAA。那么你的默认博客主页链接就应当为 https://aaaa.github.io。

部署在主页

如果你希望你的博客的链接是 https://aaaa.github.io，那么你需要做的是：

  1. 在自己的 GitHub 账号下新建一个 repo，并将名字命名为：aaaa.github.io

自定义域名

如果在部署之后，希望网址能够改成自己买的域名。那么（假定已经买好了阿里云、腾讯云之类的域名），并且设置好了域名解析（这里放一个别人的教程），那么接下来需要做的，是在当前 hexo 项目的 source 文件夹下创建名为 CNAME 的文件，其内容就是对应的新域名。

之后再进行 hexo deploy 即可生效。

其他两种方式，不管是手动在 GitHub Pages 对应的 repo 中手动添加 CNAME，还是在 Settings - Pages 中进行设置（其实就是手动创建 CNAME 的简化版），都是不行的。因为每次新的 hexo deploy 都会覆盖掉之前做的修改，导致 CNAME 文件消失。

所以，正确的做法就是我上面说的。

另外，如果你的主页已经设置了域名的跳转，然后你的另一个博客放在子目录，那么你不需要在另一个博客创建一个 CNAME 文件，就自动能对应到新的域名了。
